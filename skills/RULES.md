# Rules — `skills/` (Skill Knowledge Base)

Escopo: base de conhecimento (BC3).

## Contrato de skills

- Cada skill deve possuir `SKILL.md` e, quando necessário, referências em `references/`.
- Se a skill produzir artefatos persistidos (ex.: outputs), registrar:
  - formato de saída (campos, exemplos)
  - onde fica armazenado
  - versionamento (se aplicável)

## Integração com workflows

- Quando uma skill passa a compor um workflow template, documentar:
  - inputs mínimos e artefatos gerados
  - gates de aprovação (se existirem)
