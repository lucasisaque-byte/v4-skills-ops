# Rubrica de Qualidade — Account Manager

## Definition of Done

O trabalho do Account Manager está concluído quando:

- [ ] Tarefa classificada corretamente antes de qualquer execução
- [ ] Contexto do cliente recuperado (pasta local + docs novos)
- [ ] Documentos novos salvos na pasta do cliente
- [ ] Conflitos de contexto resolvidos pela hierarquia de prioridade
- [ ] Lacunas de contexto sinalizadas explicitamente
- [ ] Hipóteses assumidas declaradas ao usuário
- [ ] Briefing completo com todos os campos obrigatórios da skill alvo
- [ ] Output apresentado com resumo de racional (ângulo, framework, persona)
- [ ] Aprovação solicitada de forma clara e estruturada
- [ ] Feedback interpretado corretamente (ajuste vs. refação)
- [ ] Rebriefing preciso e acionável quando necessário
- [ ] Handoff reformatado antes de enviar para a próxima skill
- [ ] Nenhum output bruto passado diretamente entre skills

---

## Critérios de output forte

**Briefing forte:**
- Específico para o cliente — não genérico
- Todos os campos preenchidos ou lacunas declaradas
- Contém insight estratégico do Account Manager
- Não exige que a skill adivinhe informações ausentes
- Tom e persona claramente definidos

**Apresentação de aprovação forte:**
- Resumo do racional em 2-3 linhas (não um parágrafo)
- Framework e persona explícitos
- Instruções de resposta claras (aprovado / ajuste / refazer)
- Output apresentado de forma limpa, sem ruído

**Rebriefing forte:**
- Distingue o que muda do que permanece
- Incorpora o feedback verbatim do usuário
- Não faz suposições sobre o que o usuário quis dizer
- Instrui a skill de forma direta e acionável

---

## Critérios de output fraco

**Briefing fraco:**
- Campos genéricos ("persona: decisores de negócio")
- Informações inventadas não declaradas como hipótese
- Ausência de insight estratégico
- Tom de voz não especificado
- Objeções e diferenciais em branco

**Aprovação fraca:**
- Sem resumo do racional
- Sem instruções claras de resposta
- Output apresentado com ruído (rascunhos, notas internas)
- Pergunta de aprovação enterrada no texto

**Rebriefing fraco:**
- Trata ajuste pontual como refação total (ou vice-versa)
- Não declara o que foi mantido
- Não incorpora o feedback verbatim
- Instrução vaga para a skill ("melhore o tom")

---

## Testes de validação

Antes de considerar o trabalho concluído, responda:

1. **O briefing é específico para este cliente?**
   Se eu removesse o nome do cliente, este briefing ainda faria sentido para outro cliente? Se sim, está genérico demais.

2. **A skill alvo consegue executar sem perguntas adicionais?**
   Se a skill precisar adivinhar informações para completar a tarefa, o briefing está incompleto.

3. **O output está sendo passado bruto entre skills?**
   Se sim, o Account Manager não cumpriu seu papel de reformatação.

4. **O usuário sabe exatamente o que fazer para avançar?**
   Se a mensagem de aprovação não tiver instruções claras, a aprovação não vai acontecer de forma eficiente.

5. **O feedback foi interpretado corretamente?**
   Se o usuário pediu um ajuste e o Account Manager refez tudo (ou vice-versa), houve erro de interpretação.
