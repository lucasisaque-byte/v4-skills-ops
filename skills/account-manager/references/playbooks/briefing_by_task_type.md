# Playbook de Briefings por Tipo de Tarefa

## Princípio
Um briefing fraco gera output fraco. O Account Manager é responsável por extrair do contexto disponível as informações que cada skill precisa — sem inventar, sem generalizar, sem deixar campos críticos em branco.

Quando uma informação não existe no contexto disponível: declare a lacuna, assuma uma hipótese razoável e sinalize o risco.

---

## Briefing para Copywriting

Use para: Landing Pages, Ads, Copy standalone

Campos obrigatórios:
- **Cliente:** nome do cliente
- **Objetivo da peça:** o que essa copy deve fazer (gerar lead, agendar reunião, venda direta)
- **Tipo de peça:** LP completa / hero section / ad / email / VSL
- **Canal:** onde será publicado
- **Persona alvo:** quem vai ler (cargo, perfil, nível de consciência)
- **Membro do comitê:** Champion / CFO / CTO / Usuário técnico / Blocker
- **Nível de awareness:** não sabe do problema / sabe do problema / sabe da solução / conhece o produto / mais consciente
- **Dores prioritárias:** extraídas do DCC/UCM
- **Objeções críticas:** o que impede a compra hoje
- **Diferenciais:** o que torna esse produto único
- **Oferta:** o que está sendo vendido (produto, serviço, isca, reunião)
- **CTA:** qual ação o usuário deve tomar
- **Tom de voz:** formal / técnico-acessível / direto / consultivo
- **Restrições:** termos proibidos, promises que não podem ser feitas
- **Provas sociais disponíveis:** cases, dados, certificações, prêmios
- **Documentos-base:** quais arquivos foram usados
- **Lacunas de contexto:** o que não foi encontrado e foi assumido
- **Observações estratégicas:** insight do Account Manager sobre a tarefa

→ Template pronto: `references/templates/briefing_copywriting.md`

## Briefing para Social Media

Use para: posts avulsos, carrosseis, reels, scripts de vídeo, calendários editoriais

Campos obrigatórios:
- **Cliente:** nome do cliente
- **Objetivo do conteúdo:** engajamento / awareness / geração de lead / conversão
- **Formato:** post estático / carrossel / script de reels / script de vídeo longo
- **Rede social:** LinkedIn / Instagram / YouTube / TikTok
- **Público:** quem vai consumir o conteúdo
- **Tema:** assunto principal do conteúdo
- **Ângulo desejado:** ponto de vista, abordagem, tom
- **Tese central:** a ideia principal que o conteúdo deve comunicar
- **CTA:** qual ação o usuário deve tomar ao final
- **Restrições:** o que não pode aparecer no conteúdo
- **Contexto estratégico:** como esse conteúdo se conecta à estratégia maior
- **Referências:** exemplos de conteúdo aprovados ou de referência
- **Necessidade de deep research:** sim/não + direção da pesquisa
- **Diretrizes de marca:** tom, linguagem, elementos proibidos
- **Para calendários:** período, número e tipos de peças, eixos temáticos

→ Template pronto: `references/templates/briefing_social_media.md`

## Briefing para Designer

Use para: produção visual de qualquer peça já aprovada

Campos obrigatórios:
- **Conteúdo aprovado:** copy verbatim aprovada pelo usuário (não resumir)
- **Objetivo visual:** o que o design deve comunicar e provocar
- **Canal:** onde será publicado
- **Formato / dimensões:** 1080x1080 / 1920x1080 / stories / etc.
- **Estrutura da peça:** seções, ordem, hierarquia de informação
- **Hierarquia visual:** o que deve ter mais destaque visual
- **Tom estético:** minimalista / corporativo / vibrante / técnico / etc.
- **Referências visuais:** exemplos de design aprovados ou de referência
- **Paleta de cores:** se definida no guia de marca
- **Tipografia:** se definida no guia de marca
- **Elementos obrigatórios:** logo, tagline, URL, QR code, etc.
- **Elementos proibidos:** o que não pode aparecer
- **Observações do usuário:** feedback coletado na aprovação anterior
- **Diretrizes derivadas do conteúdo aprovado:** insights do Account Manager

→ Template pronto: `references/templates/briefing_designer.md`

## Briefing para Auditoria (Marketing ou Vendas/CRM)

Use para: análise de performance, diagnóstico de funil, revisão de CRM

Campos obrigatórios:
- **Cliente:** nome do cliente
- **Tipo de auditoria:** marketing / vendas / CRM / funil completo
- **Período analisado:** datas de início e fim dos dados
- **Dados disponíveis:** o que foi fornecido (relatório de mídia, CRM, planilha, etc.)
- **Objetivo da análise:** o que o usuário quer descobrir ou resolver
- **KPIs foco:** quais métricas são prioritárias
- **Contexto de negócio:** situação atual do cliente, histórico relevante
- **Hipóteses iniciais:** o que o usuário já suspeita
- **Output esperado:** diagnóstico / plano de ação / recomendação de skill

→ Template pronto: `references/templates/audit_crm_funnel.md`

## Briefing para Storytelling de Reunião

Use para: transformar reuniões, check-ins e alinhamentos em narrativa útil

Campos obrigatórios:
- **Cliente:** nome do cliente
- **Tipo de reunião:** check-in / alinhamento / kickoff / retrospectiva / comercial
- **Data da reunião**
- **Participantes:** quem estava presente e seus papéis
- **Objetivo da reunião:** o que se pretendia resolver ou discutir
- **Material fonte:** transcrição, ata, anotações, gravação resumida

→ Template pronto: `references/templates/checkin_storytelling.md`
